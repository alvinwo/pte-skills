# V1 Release Checklist

Use this file to decide whether `pte-skills` is ready to ship as version 1.0.0.

## Release goal
Ship a clear first version that is useful for real learners, especially:
- student visa learners
- all four communicative skills at 65+ learners
- all four communicative skills at 79+ learners

## Must-have checklist
- [x] current PTE question-type audit
- [x] practical score-priority matrix
- [x] target-score planning logic
- [x] scenario-based plans for student visa / all four communicative skills at 65+ / all four communicative skills at 79+
- [x] daily practice system
- [x] vocabulary strategy
- [x] high-yield vs low-yield question guidance
- [x] official mock-test guidance
- [x] visa English requirement reference with source links
- [x] core skill pages for RA / RS / DI / RL / SGD / RTS / WFD / reading blanks
- [x] a Start Here onboarding page
- [ ] README updated from draft framing to release framing
- [ ] one example file with concrete learner plans
- [ ] decide whether one more writing-oriented skill is required before v1
- [ ] choose and add a real license

## Nice-to-have but not blocking
- [ ] question banks
- [ ] benchmark answer libraries
- [ ] mock-test result analysis workflows
- [ ] complete coverage of every PTE question type
- [ ] automated scoring logic
- [ ] scenario-specific weekly planner templates

## Release decision rule
Version 1.0.0 is ready when:
1. a new learner can find the right entrypoint quickly
2. the repo can produce a clear plan for student visa / all four communicative skills at 65+ / all four communicative skills at 79+ users
3. the repo gives strong guidance on what to prioritize and what to deprioritize
4. the repo explains what to do every day and every week
5. the repo has enough high-value task coverage to be genuinely useful even without full task coverage

## Recommended final pre-release pass
Before tagging v1.0.0:
- simplify README language for first-time visitors
- add one example plans file
- decide the v1 cut line clearly
- check for repeated or overlapping files that can be linked more clearly
- make sure the repo description matches the real scope
